xtuml2masl

This command line tool converts an xtUML package into an equivalent MASL domain or MASL project depending on parameter settings

Syntax[a]

xtuml2masl         [ -i <input.sql > -d <package  name> -o < output directory > ]  |

 [-i <directory path > -p <package name> -o < output directory > ]

Where

-i                                 indicates the xtUML  project

<input.sql>        is an absolute or relative path ending at the sql file of an xtUML project

-d                                 indicates the xtUML package will be exported as a MASL domain

-p                                 indicates the xtUML package will be exported as a MASL project

<package name>                 is the name of an xtUML package within the project

<output directory>        is an absolute or relative path to the directory storing the generated MASL

Notes

  • The tool is located in the subdirectory <BridgePoint Home>/tools/masl
  • Progress and Info messages are reported to stdout while error messages are reported to stderr.
  • Invoking the tool with missing parameters or syntax errors produces a message indicating proper usage.

Examples

xtuml2masl -i GPSwatchMASL/Tracking/gen/codegen/input.sql -d Tracking -o TrackingMASL

xtuml2masl -i GPSwatchMASL/UI/gen/codegen/input.sql -d UI -o U_MASL

[a]This syntax specification does not align with the one in the readme.md file provided by Levi in the Testing and Documentation chat.  Specifically, it is missing the -o parameter which allows the user to specify the destination for the output.
